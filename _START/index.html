What is CSS Grid

CSS Grid Layout also known simply as "Grid" is a two-dimensional layout engine that is completely revolutionizing the way we handle our macro and micro layouts in CSS. There has been a steady evolution in the way we manage layouts in CSS. We started with tables, then floats, then positioning. None of which were very efficent, however flexbox has been the first solid step into proper layout management. However, flexbox is only one-dimensional, this is where grid comes in. It can solve what flexbox can't by handling alignments and sizing in two dimensions. Now this doesn't mean grid is a replacement for flexbox, flexbox and grid actually work really well together and will be the future of web based layouts.

Terminology

When learning CSS Grid there will be a few new terms thrown around you will hear. Before we dive into the code let's just quick go over some new terminology.

Grid Container

The Grid Container is the element with the display: grid property value applied to it. In the example below the .grid div would be the Grid Container.

Example
                    
                      <div class="grid">
                        <div class="child"></div>
                        <div class="child"></div>
                        <div class="child"></div>
                      </div>
                    
                    
                    .grid {
                      display: grid;
                    }

Grid Item

The Grid Item is any direct child of a Grid Container. So you can see in the example below, only the elements with a class .is-grid-cell are grid cells. That is strictly because it's a direct child of the Grid Container.

Example
                    
                        <div class="grid">
                          <div class="is-grid-cell"></div>
                          <div class="is-grid-cell">
                              <p>Hello world</p>
                          </div>
                          <div class="is-grid-cell"></div>
                        </div>
                    
                    
                      .grid {
                        display: grid;
                      }

Grid Line

Grid Lines are the dividing lines that construct the grid. They can also be view as the "column lines" and the "row lines" that sit on the side of a row or column. Below you can see a red line as an example of a Grid Line.

Example

Grid Cells

Grid Cells are simply the space between two adjacent rows and two adjacent columns. They can have all sorts of sizes and positioning based on the layout of your grid. In this example the red sqaure is a Grid Cell located on the row lines 1 and 2, and the column lines of 2 and 3.

Grid Track

A Grid Track essentially is the space between two adjacent grid lines. Another way you can look at them is as series of grid cells spanning over mulitple columns or rows. Below you can see a Grid Track located between the row lines 1 and 4 and column lines 1 and 2.

Grid Area

Grid Area is the total space which is surrounded by 4 different grid lines. Grid areas can have a various number sizes and positions but as a whole, it's any number of grid cells surrounded by four grid lines. In the example below you can see a Grid Area located between row grid lines 1 and 3, and column grid lines 1 and 3.

Browser Support

Other than good old Internet Explorer, CSS Grid is unprefixed in all the major browser Safari, Chrome, Opera, Firefox and Edge. This means that if you write some Grid Layout code in Firefox, it should work in the same way in Chrome. It is no longer an experimental specification, and you are safe to use it in production!

It should be noted that the original implementation of CSS Grid was on Internet Explorer 10. This early implementation didn't contain all of the properties and values that the current specification will have. There is also a fair differences between what shipped in Internet Explorer 10 and the current specification, even where the properties and values appear the same. This early specification is also the version of CSS Grid implemented in Edge up to version 15.

Columns & Rows

So you can't have a CSS Grid without columns and rows, but how do you use them for creating different kinds of layouts. We it's simple, all you need to do is specify the number of columns or rows you would like your grid to have. For example if you are wanting a three column grid, you can use the grid-template-columns CSS property to specify three columns in your grid. You also have tons of flexibility with the sizing of those columns. The same rules apply for rows, you can use the grid-template-rows property to specify the desired number and size of rows you would like to have in your grid.

There are countless different variation of values you can use when creating your columns and rows but let's go over the most frequently used methods below. We will touch on some more complicated examples later on in the course.

Ok let's start with an easy example by creating simple three by three grid.

Practice Step
Solution
                        
                            .grid {
                                display: grid;
                                grid-template-columns: repeat(3, 1fr);
                                grid-template-rows: repeat(3, 150px);
                            }

Implict vs Explict

So this is just a quick note regarding how we are sizing our grid. You will often hear the term implict or explict but what does that actually mean? Well all it mean is, are we setting fixed sizes to our grid or not. So for example, if we set our grid columns to have a fixed width of say 200px then we now have explicitly defined what the columns. However, we have not defined a size to our rows which would make our rows implicit. So if you have an implict row or column then the browser will decide how to render them based on the elements around them as well as the content within the grid cells. By default implict rows and columns are set to stretch.

One useful thing to note is that you can see your implicit and explicit lines that are defined, by using your browser's inspect element. Most browsers today support inspecting grid elements, however, at the time of this lesson Firefox by far has the best CSS Grid support. When inspecting a grid element you will see a series of lines over your grid. The solid lines represent where the explicit grid is starts or stops. The darker dashed line represents a row or column that has explicitly been defined. Lastly a lighter dashed line represents an implicity defined row or column.

Important note about the explicit grid lines. The explicit grid lines doesn't necessarily mean it's going to wrap the entire grid like a box. It's only going to wrap the explicitly specified size of the grid. So for example if you defined three columns and two rows in your grid, yet you have nine grid cells. The explicit grid will only wrap around the first six grid cells because you only defined a 3x2 grid. The following grid cells(seven, eight and nine in this example) will implicity wrap below the explicit grid.

Practice Step
Solution
                        
                            .grid {
                                display: grid;
                                grid-template-columns: repeat(3, 1fr);
                                grid-template-rows: 100px 100px;
                                grid-auto-rows: 200px;
                            }

Grid Gaps

The grid-gap CSS property which is now simply called gap will set gutters between rows and columns. You can also use the row-gap and column-gap properties but grid-gap is simply the shorthand. It's important to note that grid-gap is now just called gap, however, not all browsers support the new naming convention so its best to include both as a fallback.

Practice Step
Solution
                        
                            .grid {
                                display: grid;
                                grid-template-columns: repeat(3, 1fr);
                                gap: 10px;
                              }

minmax()

Another import part of the CSS Grid engine is the minmax() function. This will take in two arguments, the first is the minimum size of the track and the second the maximum size. The paramteters can be a number of different value types such as px, em, rem, %, fr, or it can also be auto, which allows the track to grow/stretch based on the size of the content. This will become very useful to you when creating flexible layouts because you can make your content scale with the screen while at the same time preventing the you elements from closing in on themselves while they shrink. It also will be replacing a lot of the media queries you would have been using in the past due to it's ability to scale at different sizes.

You will also likely be using the auto-fill and auto-fit values inside your minmax() function so that you don't need to explicitly define column or row sizes.

Practice Step
Solution
                      
                        .grid {
                            display: grid;
                            grid-template-columns: repeat(3, minmax(50px, auto));
                        }

Alignment & Justifying

The next thing you will find yourself using a lot is aligning and justifying the contnent inside your grid. You can either apply a rule of alignment to all the grid cells that are inside your grid by using the align-items property on the grid itself. Otherwise, if you want to just align maybe a particular grid cell you can do so by using the align-self property on the grid cell you want to target.

When you are aligning or justifying the content inside a grid it is important to know that aligning will be up and down, and justifying with be left and right.

In the next example let's look at some different ways we can align our grid-cells and create some pretty cool layouts.

Practice Step
Solution
                        
                            .grid {
                                display: grid;
                                grid-template-columns: repeat(3, 1fr);
                                grid-template-rows: 150px 150px 150px;
                                align-items: center;
                                justify-items: center;
                            }
                            .grid div:nth-child(2) {
                                align-self: end;
                            }
                            .grid div:nth-child(8) {
                                align-self: start;
                            }

Positioning Items

So we know how to create a grid, set our columns and rows within the grid, align our grid cells and manipulate the sizes of our grid lines. The next step is to understand how to start placing elements in our grid. This is the first useful feature we will learn for building different layouts and is actually pretty simple to pick up. In order to position elements within the grid there are two properties you will be using, grid-column and grid-row. There are a few others such as grid-row-start, grid-row-end, grid-column-start and grid-column-end, however , grid-column and grid-row are two useful shorthands for all those properties.

We quickly went over grid lines at the start of the series but let's briefly go over one more thing around grid lines. There are column grid lines and row grid lines and each grid line has an index which is used for referencing it's points in the grid. Grid lines start counting at one and increment by one after each row or column. One thing to note is the last grid line is always the amount of columns or rows + one. So for example a 3x3 grid would have four column grid lines and four row grid lines. See the example below for a better picture.

When positioning elements in the grid we use grid-column to position the specific element between two column grid lines. It takes two values seperated by a "/". The first value is the start point and the second value is the end point. For example grid-column: 1/3 is telling the element to span the width of the first column and the the second column. This is because the third column grid line is after the second column but before the third column, so the flex-track(element we are targeting) is only told to span up until the start of the third column. The same rules apply for the grid-row property but instead of columns its spanning rows.

One fun little note is you can actually even name grid lines with the grid-template-columns and grid-template-rows properties. All you need to do is grid-template-rows: [line-1-start] 1fr [line-2-start] 1fr [line-2-end];. Then you can just referencing those line names instead of the line indexes. Just like grid-row: line-1-start/line-2-end.

Now let's actually play around with positioning and you can see how powerful it can be. It makes what would be complex layouts in the past very easy.

Practice Step
Solution
                        
                            .positioning .grid {
                              display: grid;
                              grid-template-columns: repeat(3, 1fr);
                            }
                            .positioning .grid > div:nth-child(1) {
                                grid-row: 1/2;
                                grid-column: 1/2;
                            }
                            .positioning .grid > div:nth-child(2) {
                                grid-row: 2/3;
                                grid-column: 2/3;
                            }
                            .positioning .grid > div:nth-child(3) {
                                grid-row: 3/4;
                                grid-column: 3/4;
                            }

12 Column Layout

Ok so now that we have an understanding of how to build a proper grid layout and align our grid cells within the grid, let's now look at a simple example of where we could use this in the real world. In this step we are going to go over how to position our grid cells throughout the grid in order to create a simple wireframe design.

Practice Step
Solution
                        
                            .grid {
                                display: grid;
                                grid-template-columns: repeat(12, 1fr);
                                grid-auto-rows: minmax(100px, auto);
                                grid-gap: 10px;
                            }
                            header {
                                grid-column: 1/13;
                            }
                            main {
                                grid-column: 7/13;
                                grid-row: 2/4;
                            }
                            aside {
                                grid-column: 1/7;
                                grid-row: 2/3;
                            }
                            section {
                                grid-column: 1/13;
                                grid-row: 4/6;
                            }
                            nav {
                                grid-column: 1/7;
                                grid-row: 3/4;
                            }
                            footer {
                                grid-column: 1/13;
                                grid-row: 6;
                            }

Overlapping

Another powerful feature of CSS Grid is the ability to easily to overlap elements. Traditionally you would need to use positions which is normally fine most of the time, however, sometimes it can be a problem. For example if you are using a parallax library, having an absolute positioned element can have conflicts with the layout around it. Another issue is in complex layouts there may be wrappers around an item you want to put behind something, however if one of those wrappers or parents may have a position: relative value it will conflict with the positioning you're tring to achieve.

With CSS Grid you can easily overlap elements by assigning their coordinates within the grid. If your coordinates are overlapping or set in the same place as other grid cells, they would effect each other, other than in the z axis.

Let's look at a simple example of where overlapping could be useful in CSS grid.

Practice Step
Solution
                        
                            .overlapping {
                                max-width: 500px;
                            }
                            .overlapping .grid {
                                display: grid;
                                height: 500px;
                                grid-template-columns: repeat(4, 1fr);
                            }
                            .overlapping .grid > div:nth-child(1) {
                                grid-column: 1/7;
                                grid-row: 1/4;
                            }
                            .overlapping .grid > div:nth-child(2) {
                                grid-column: 2/4;
                                grid-row: 2/3;
                            }

Grid Areas

This might be the coolest part of CSS grid. It's probably the most progressive feature of the CSS grid engine and allows us to make complex layouts while still keeping our CSS very readable. So what is this magic? Well the grid-area property specifies a grid item's location and size within a grid layout. So right now you might be thinking, well this sounds just like the grid positioning we were doing earlier in the 12 column example and it is similar. However, the grid-area property can also be used to assign a name to a particular element in a grid(grid track). Named grid items can then be referenced to by the grid-template-areas property of the grid container.

This still might sound a little confusing, so let's just jump into an example and you will get it.

Practice Step
Solution
                        
                            .grid {
                                display: grid;
                                gap: 10px;
                                grid-template-columns: repeat(4, 1fr);
                                grid-auto-rows: minmax(100px, auto);
                                grid-template-areas:
                                "header header header header"
                                "aside aside main main"
                                "nav nav main main"
                                "section section section section"
                                "section section section section"
                                "footer footer footer footer";
                            }
                            
                            .grid header {
                                grid-area: header;
                            }
                            .grid main {
                                grid-area: main;
                            }
                            .grid aside {
                                grid-area: aside;
                            }
                            .grid section {
                                grid-area: section;
                            }
                            .grid nav {
                                grid-area: nav;
                            }
                            .grid footer {
                                grid-area: footer;
                            }

Responsive Layouts

There is a good chance you are already familiar with CSS media queries, but you may not realize the potential they have with the CSS grid engine. In this section we will go over making a responsive layout that can adapt to any screen size.

let's say we are going to minic our previous page layout above in the grid-area section. However, this time around on mobile we want everything to be full width, but maybe we don't the same stacking order we have on desktop. Instead we want the nav bar to be above the main section and we want the main section to be above the aside element.

When looking at our HTML markup we can quickly see our current structure doesn't fit this order on mobile. Tranditional this would be an issue, we would either need to make sure we have no wrapper elements around any of these elements which would be unlikely if we ended up wanting to align or justify our content. Then use the order property in flexbox. Or we would need to duplicate our markup and then need to display: none one or the other depending if were on mobile or desktop. Let me just say that, that should never be the case even though I see it all the time!

Now let's see how we would manage this with CSS grid and the grid-area property. I am sure you will love this.

Practice Step
Solution
                        
                            .grid-areas {
                                max-width: 500px;
                              }
                              .grid-areas .grid {
                                display: grid;
                                gap: 10px;
                                grid-template-columns: repeat(4, 1fr);
                                grid-auto-rows: minmax(100px, auto);
                                grid-template-areas:
                                  "header header header header"
                                  "aside aside main main"
                                  "nav nav main main"
                                  "section section section section"
                                  "section section section section"
                                  "footer footer footer footer";
                              }
                              @media screen and (max-width: 960px) {
                                .grid-areas .grid {
                                  grid-template-areas:
                                    "header header header header"
                                    "nav nav nav nav"
                                    "main main main main"
                                    "aside aside aside aside"
                                    "section section section section"
                                    "section section section section"
                                    "footer footer footer footer";
                                }
                              }
                              .grid-areas .grid header {
                                grid-area: header;
                              }
                              .grid-areas .grid main {
                                grid-area: main;
                              }
                              .grid-areas .grid aside {
                                grid-area: aside;
                              }
                              .grid-areas .grid section {
                                grid-area: section;
                              }
                              .grid-areas .grid nav {
                                grid-area: nav;
                              }
                              .grid-areas .grid footer {
                                grid-area: footer;
                              }
